PushScripts

Ship code faster with AI-powered git commands. Stop writing commit messages manually - let AI handle the documentation while you focus on coding.

Ready to start? Choose global installation for personal use or project integration for team workflows.

Global Installation

Use AI-powered commits in any git repository.

RECOMMENDED 
# Install globally
pnpm add -g pushscripts

# Use 'push' to commit and push
push

# Or 'commit' to only commit

Project Integration

Set up smart commits for your team's workflow.

# Install as dev dependency
pnpm add -D pushscripts

# Copy environment file
cp .env.example .env

# Add to package.json scripts
{
  "scripts": {
    "push": "node src/core/cli.js",
    "commit": "node src/core/cli.js"
  }
}

# Use locally with pnpm
pnpm push    # commit and push
pnpm commit  # commit only

Quickstart

Get started with AI-powered commit messages in minutes.

Before you begin

Make sure you have your Groq API key ready (any model should work). You can get one from the Groq Console.

# Install globally
pnpm add -g pushscripts

# Set up your API key
echo "GROQ_API_KEY=your-api-key" > ~/.pushscripts-rc

# Use in any git repository
git add .
push

Configuration

Customize PushScripts to match your team's workflow and handle different environments securely.

Environment
# Required: Your Groq API key for AI commit message generation
GROQ_API_KEY=your-api-key-here

# Optional: Default git branch (defaults to 'main')
GIT_DEFAULT_BRANCH=main

# Optional: Customize commit message settings
COMMIT_MESSAGE_TEMPERATURE=0.3
COMMIT_MESSAGE_MODEL=llama2-70b-chat

# Optional: Enable debug mode for detailed logging
DEBUG=pushscripts:*
Security Best Practices

Never commit your API key. For global installation, use ~/.pushscripts-rc. For projects, use .env and ensure it's in .gitignore. For CI/CD, use secure environment variables.

Debug Mode

Enable debug mode by setting DEBUG=pushscripts:* to see detailed logs about git operations, API calls, and commit message generation.

Node.js
// Using in your own Node.js script
const GitPushAI = require('pushscripts');

async function handleGitCommit() {
  try {
    // Initialize with API key
    const gitPush = new GitPushAI(process.env.GROQ_API_KEY);

    // Check for sensitive files first
    const sensitiveFiles = gitPush.checkSensitiveFiles();
    if (sensitiveFiles.length > 0) {
      console.error('Found sensitive files:', sensitiveFiles);
      return;
    }

    // Get git status
    const changes = gitPush.getGitStatus();
    if (changes.length === 0) {
      console.log('No changes to commit');
      return;
    }

    // Generate commit message with error handling
    try {
      const message = await gitPush.generateAICommitMessage(changes);
      console.log('Generated message:', message);
    } catch (apiError) {
      console.error('AI generation failed, using basic message');
      const basicMessage = gitPush.generateBasicCommitMessage(changes);
      console.log('Basic message:', basicMessage);
    }
  } catch (error) {
    console.error('Error:', error.message);
  }
}

## Team Setup

2. Copy .env.example to .env
3. Add your GROQ_API_KEY
4. Use 'pnpm push' instead of 'git push'
